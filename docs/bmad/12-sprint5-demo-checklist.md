# Sprint 5/6 演示检查清单

## 演示地址

- 学员端：`http://127.0.0.1:4317/v3.2.html`
- 后台端：`http://127.0.0.1:4317/admin.html`

必须使用 HTTP 地址验收。`file://` 不能验证注册、登录、题库提交、Profile 和后台接口。

## 五分钟演示路径

1. 在 V3.2 注册一个学员。
2. 查看 Today 是否推荐晋级相关课程或题组。
3. 打开课程详情并标记课程完成。
4. 进入题库，按课程或知识域筛选并完成一组题。
5. 打开 Profile，查看能力画像、晋级状态和最近晋级历史。
6. 使用管理员账号登录后台。
7. 在后台总览查看该用户的课程、题目、风险和晋级摘要。
8. 打开用户详情，确认晋级路径、缺失课程、题目正确率和晋级历史。

## UI 验收

- 学员端检查 390px、430px、768px。
- 后台端检查移动端和桌面端。
- 没有明显横向溢出。
- 关键按钮可点击，触控区足够。
- MC 迷彩背景不影响文字可读性。
- 空状态说明下一步动作。
- V3.2 不显示课程图片。

## 数据验收

- Profile、Today、后台详情使用同一套八个能力域。
- 弱项建议说明样本量或正确率依据。
- 新用户不展示伪造雷达。
- 有答题记录的用户展示真实能力数据。
- 完成阶段课程且课程题正确率大于 90% 后自动晋级。
- 晋级写入 `level_history`。
- 后台保持只读。

## 质量门禁

演示前运行：

```bash
npm run check
npm run test:api
npm run test:perf
```

必须覆盖：

- 课程内容：6 阶段、30 周、61 课。
- 题库：覆盖课程题和情景题。
- Profile API 返回 8 个能力域。
- 自动晋级：90% 不晋级，91% 晋级。
- 性能 smoke 能处理 100 用户和 30,000 条答题记录。

## 已知限制

- 教官端运营、装备场地、成就体系不在当前范围。
- 后台不提供编辑、删除、重置、导出。
- SQLite 使用 Node 内置 `node:sqlite`，experimental warning 属于预期。
- 历史 V2/V3/V3.1 文件保留，但不作为推荐入口。
