# 自动晋级规则

## 规则原则

晋级只看两个真实数据源：

- 用户完成的课程。
- 对应阶段课程题的最新答题正确率。

不纳入人工评估、AAR、装备、安全违规、互评、成就或教官审核。

## 阶段规则

| 晋级 | 课程范围 | 题目范围 | 条件 |
|---|---|---|---|
| L0 -> L1 | W1-W6 | W1-W6 课程题 | 课程完成且正确率 > 90% |
| L1 -> L2 | W7-W10 | W7-W10 课程题 | 课程完成且正确率 > 90% |
| L2 -> L3 | W11-W14 | W11-W14 课程题 | 课程完成且正确率 > 90% |
| L3 -> L4 | W15-W18 | W15-W18 课程题 | 课程完成且正确率 > 90% |
| L4 -> L5 | W21-W24 | W21-W24 课程题 | 课程完成且正确率 > 90% |
| L5 -> L6 | W25-W30 | W25-W30 课程题 | 课程完成且正确率 > 90% |

## 计算口径

- 只统计 `questionType/type = course` 的题目。
- 只统计 `lessonMapping` 落在当前阶段的题目。
- 同一道题多次作答只取最新一次。
- 阶段课程题答题数为 0 时不能晋级。
- 正确率必须大于 90%，等于 90% 不晋级。
- 暂无对应题目的课程不影响课程完成条件，但题目条件仍要求该阶段至少有答题记录。

## 触发时机

- 标记课程完成后检查。
- 提交题组后检查。
- 读取 `/api/me`、`/api/profile`、后台用户列表、后台用户详情时兜底检查。

## 数据记录

- 当前等级写入 `users.level`。
- 每次晋级写入 `level_history`。
- 晋级历史记录 from、to、原因、课程完成数、题目正确数、题目总数、正确率和时间。
