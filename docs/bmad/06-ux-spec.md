# UX Spec

## Mobile Rules

- First screen after login answers "what should I continue today?"
- Course map is a phase path, not a static catalog.
- Question bank starts with three decisions only: quick practice, course practice, knowledge-domain practice.
- Answering always shows current progress and a reachable submit action.
- Results show score, pass/fail, wrong question numbers, and tags.
- Profile explains data source and never fakes conclusions.

## Empty States

- No questions: offer return to all questions or change filter.
- No profile data: prompt user to complete one course or one question set.
- No admin users: show an empty table with no fake demo rows.

## Admin UX

Admin prioritizes scan speed: user list first, detail below, no operational controls beyond refresh and viewing detail.
